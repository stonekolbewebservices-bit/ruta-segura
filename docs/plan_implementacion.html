Ruta Segura - Plan de Implementación

Descripción del Objetivo

Crear una aplicación web que ayude a los viajeros a planificar rutas seguras. La app mostrará un mapa estilizado donde las rutas se colorean según métricas de seguridad (datos simulados/reales). Incluirá funciones para ver estadísticas detalladas de delincuencia en ciudades a lo largo de la ruta. El objetivo es un MVP con estética "premium" (tipo videojuego/Strava) autosostenible mediante AdSense.

Revisión del Usuario Requerida

IMPORTANTE - Fuentes de Datos: Los datos reales de crimen (secuestros, etc.) son difíciles de obtener gratuitamente y en tiempo real vía API pública. Para este MVP, usaremos Datos Simulados (Mock Data) para demostrar la funcionalidad y la interfaz. Podremos explorar APIs reales (gobierno, etc.) en el futuro.
Motor de Mapas: Para mantener el costo en $0, usaremos Leaflet con datos de OpenStreetMap y una capa de estilo "Dark Mode" personalizada. Evitaremos Google Maps API por ahora ya que requiere tarjeta de crédito y tiene costos.

Cambios Propuestos

Tecnología Principal

  • Framework: React (Vite) para rendimiento.
  • Estilo: Tailwind CSS para diseño moderno y rápido.
  • Mapas: react-leaflet + leaflet.
  • Rutas: Cálculo de rutas simulado o usando OSRM demo para el prototipo.

Arquitectura

Diseño (UI/UX)

  • Layout Principal: Mapa a pantalla completa como fondo.
  • Estilo de Mapa: "Dark Matter" o "Cyberpunk" (tonos oscuros y neón).
  • Indicadores: Rutas verdes (seguras) a rojas (peligrosas).

Capa de Datos

  • Servicio Mock: Funciones que devuelven puntuaciones de seguridad aleatorias pero consistentes para las demos.
  • Tarjetas de Estadísticas: Paneles deslizables con datos de homicidios, asaltos, etc.

Estructura de Archivos

src/
  components/
    Map/      (Lógica del mapa)
    UI/       (Buscadores, Tarjetas de info)
  services/   (Datos simulados, Rutas)
  assets/     (Iconos, Estilos)

Plan de Verificación

  1. Abrir la app en el navegador.
  2. Ingresar Origen (ej. CDMX) y Destino (ej. Guadalajara).
  3. Verificar que la ruta se dibuja y cambia de color.
  4. Hacer click en los marcadores de ciudades para ver las "stats" de seguridad.
  5. Probar la vista en modo móvil (responsive).
